# 🚀 מדריך פריסה לאינטרנט

המטרה: שהאתר יהיה זמין בכתובת קבועה שחברים יוכלו להיכנס אליה מכל מקום,
עם שליחת מייל אמיתית.

הפריסה דורשת את החשבונות שלך (GitHub + שירות אירוח), ולכן את השלבים האלה
מבצעים דרך הדפדפן. הקוד כבר מוכן ומחויב ב-git מקומית.

---

## שלב 1 — הכנת Gmail לשליחת מייל (5 דקות)

1. היכנס ל-Google → הפעל **אימות דו-שלבי** (2-Step Verification):
   <https://myaccount.google.com/security>
2. צור **App Password**: <https://myaccount.google.com/apppasswords>
   - תקבל קוד בן 16 תווים (למשל `abcd efgh ijkl mnop`).
   - שמור אותו — תזין אותו בשירות האירוח (לא בקוד!).

> חשוב: זו **לא** סיסמת הג'ימייל הרגילה שלך, אלא סיסמת אפליקציה ייעודית.

---

## שלב 2 — העלאת הקוד ל-GitHub

1. צור חשבון/התחבר ב-<https://github.com>.
2. צור ריפו חדש (ריק, בלי README): למשל בשם `mondial-2026`.
3. ב-GitHub יוצג לך כתובת הריפו. בטרמינל, בתיקיית הפרויקט, הרץ:

```bash
git remote add origin https://github.com/<USERNAME>/mondial-2026.git
git branch -M main
git push -u origin main
```

(GitHub יבקש התחברות — אפשר דרך הדפדפן.)

## שלב 3 — פריסה ב-Render (מומלץ, הכי פשוט)

1. היכנס ל-<https://render.com> והתחבר עם GitHub.
2. לחץ **New +** → **Blueprint**, ובחר את הריפו `mondial-2026`.
   Render יזהה אוטומטית את הקובץ `render.yaml`.
3. במסך ההגדרות, הזן את משתני הסוד (Environment):
   - `SMTP_USER` = כתובת הג'ימייל שלך
   - `SMTP_PASS` = ה-App Password מהשלב הראשון (16 תווים, בלי רווחים)
   - `MAIL_FROM` = כתובת הג'ימייל שלך
   - `ADMIN_EMAILS` = המייל שלך (כבר מוגדר כברירת מחדל)
   - `FOOTBALL_API_KEY` = (אופציונלי) מפתח API לעדכון תוצאות אוטומטי — ראו שלב 4
4. לחץ **Apply / Create**. אחרי דקה-שתיים תקבל כתובת כמו
   `https://mondial-2026.onrender.com` — זה הלינק שתשלח לחברים.

### לגבי שמירת נתונים (חשוב!)
- ה-`render.yaml` מוגדר עם **דיסק קבוע** (`/data`) ותוכנית `starter` (~7$ לחודש),
  כדי שהניחושים והדירוג **לא יימחקו** בעדכונים. מומלץ לטורניר אמיתי.
- רוצה **בחינם**? ערוך את `render.yaml`: מחק את השורות `plan: starter` ואת בלוק
  `disk:` (וגם את `DB_PATH`). שים לב — בתוכנית החינמית מסד הנתונים **מתאפס**
  בכל פריסה מחדש ולפעמים אחרי חוסר פעילות. מתאים רק לניסוי קצר.

## שלב 4 — עדכון תוצאות אוטומטי (אופציונלי, מומלץ)

במקום להזין תוצאות ידנית, אפשר לחבר API שמושך את התוצאות הרשמיות לבד.

1. הירשמו (חינם) ב-<https://dashboard.api-football.com> וקבלו `API Key`.
   המסלול החינמי מאפשר כמות קריאות יומית — מספיק לעדכון אחת לכמה דקות.
2. בהגדרות הפריסה הוסיפו:
   - `FOOTBALL_API_KEY` = המפתח שקיבלתם
   - (ברירות מחדל כבר מוגדרות: `WC_LEAGUE_ID=1`, `WC_SEASON=2026`, `POLL_MINUTES=10`)
3. בפאנל הניהול → "עדכון תוצאות אוטומטי":
   - לחצו "📋 הצג מזהי משחקים מה-API" כדי לראות את רשימת המשחקים והמזהים.
   - לכל משחק שיצרתם, הדביקו את ה"מזהה" בשדה "מזהה משחק ב-API".
4. מכאן זה אוטומטי: כל `POLL_MINUTES` דקות המערכת בודקת אילו משחקים הסתיימו,
   שולפת את **תוצאת 90 הדקות** ואת **מי שעלה** (כולל פנדלים), ומחשבת ניקוד לכולם.
   אפשר גם ללחוץ "⚡ סנכרן תוצאות עכשיו" בכל רגע.

> אם המפתח לא מוגדר — הכל עובד רגיל בהזנה ידנית. ההזנה הידנית תמיד זמינה גם
> כשה-API פעיל (גיבוי במקרה שמשחק לא מסונכרן).

## חלופות לאירוח (אם מעדיף)

## אחרי הפריסה
1. היכנס לאתר עם המייל שלך (תקבל קוד אמיתי במייל) — אתה אדמין.
2. בטאב **ניהול** הזן את משחקי שמינית הגמר.
3. שלח את הלינק לחברים. כל אחד נרשם עם המייל שלו.
4. אחת ל-12 שעות — עדכן תוצאות בפאנל הניהול, והדירוג יתעדכן לכולם.

צריך עזרה בשלב מסוים? אמור לי באיזה שלב אתה ומה אתה רואה, ונתקדם יחד.
